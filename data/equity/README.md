# `data/equity/` — vendored equity-driver feature series

Weekly series backing the four equity analyst personas (`sector_breadth`,
`vol_regime`, `positioning`, `risk_appetite`). Loaded by
`src/data/equity_local.py` and referenced by `personas/<driver>.yaml` through the
closed feature-op vocabulary (`op: level`).

## Provenance

Generated by `macrollm.scripts.export_equity_series` in the sibling **macro-llm**
repo (round 7, `r8-target-handoff` branch) from the frozen r7 feature grid. One
CSV per feature, two columns `date,<SERIES_ID>`, 752 Fridays 2012-01-13 →
2026-06-05.

## Point-in-time contract (important)

Each series is indexed by its **decision Friday** — the date the value was
knowable at that week's close. The COT publication lag (`usable_from = report
Tuesday + 10 days`) is **already baked in** upstream. Therefore
`equity_local.load_series` applies **no** `PUBLICATION_LAG_DAYS` shift — unlike
`fred_local`, whose FRED CSVs are observation-dated and shifted on load. Do not
add an entry for any `EQ_*` id to the FRED lag table.

## Why vendored features, not raw inputs

The op vocabulary (`src/layered/features/ops.py`) deliberately cannot express
cross-sectional breadth, rolling-std realized vol, or 52-week z-scores — a spec
cannot fit or standardize. Those features are computed in macro-llm and shipped
here as first-class series so a persona reads them with `op: level`.

## Series

| series id | feature | driver |
|---|---|---|
| EQ_BREADTH_PCT | share of sectors above their 26-wk mean | sector_breadth |
| EQ_CYCDEF_4W | cyclical − defensive 4-wk basket return | sector_breadth |
| EQ_SECTOR_DISP | cross-sector 4-wk return dispersion | sector_breadth |
| EQ_VIX | VIX level | vol_regime |
| EQ_VRP | VIX/100 − 8-wk realized vol | vol_regime |
| EQ_VIX_CHG4 | 4-wk VIX change | vol_regime |
| EQ_AMGR_Z | asset-manager net, 52-wk z | positioning |
| EQ_AMGR_CHG4 | asset-manager 4-wk net change (z units) | positioning |
| EQ_LEV_Z | leveraged-fund net, 52-wk z | positioning |
| EQ_LEV_CHG4 | leveraged-fund 4-wk net change (z units) | positioning |
| EQ_CG_CHG8 | copper/gold ratio, 8-wk change | risk_appetite |
| EQ_SLOPE_BP | 10y − 2y curve slope (bp) | risk_appetite |
| EQ_DXY_CHG8 | dollar index, 8-wk change | risk_appetite |

Values are **raw** (unrounded). macro-llm's r7 render applied era-blunting
rounding for the memorization control; that is not reproduced here, so a **live**
persona run is a new experiment, not the validated r7 vector signal. The
validated signal lives in `../equity_replay/` and is consumed by
`CsvReplayAnalyst`.
